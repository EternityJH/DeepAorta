# Dependency Inventory

*Read this in other languages: [繁體中文](docs/zh-TW/dependency_inventory.md).*

## 1. Built-in 3D Slicer Modules
DeepAorta relies on the following core modules inside 3D Slicer:
- **Volumes**: Used to manage and convert volume data (`CreateLabelVolumeFromVolume`, `CreateScalarVolumeFromVolume`).
- **Segmentations**: Processes the segmentation results generated by models (`ExportVisibleSegmentsToLabelmapNode`, `ImportLabelmapToSegmentationNode`).
- **Markups**: Handles control point data for the centerline endpoints (`vtkMRMLMarkupsFiducialNode`, `vtkMRMLMarkupsCurveNode`).
- **Plots**: Generates and visualizes quantitative data such as the area/diameter charts (`vtkMRMLPlotSeriesNode`, `vtkMRMLPlotChartNode`).
- **Tables**: Stores statistical data into tabular formats (`vtkMRMLTableNode`).

## 2. Extensions / External Modules
To use DeepAorta, users must manually install these extensions via the Slicer Extension Manager:
- **TotalSegmentator**: (Option 1) Fully automated and highly precise aorta segmentation.
- **MONAIAuto3DSeg**: (Option 2) Provides another MONAI-based aorta segmentation fallback via the `aorta-v1.1.0` model.
- **SlicerVMTK (Vascular Modeling Toolkit)**: Used to extract the aorta centerline spanning from the first to the furthest bounding endpoint.
- **Sandbox (or CurvedPlanarReformat)**: Grants the required `CurvedPlanarReformat` functionality, allowing the script to flatten the curved aorta image/labelmap natively.

## 3. Python Packages
DeepAorta employs built-in `pip_install` commands for required libraries if missing upon startup:
- **Essentials Installed Automatically**:
  - `scikit-image`: Used for image smoothing and convex hull measurements (`convex_hull_image`, `morphology`).
  - `pandas`: Responsible for organizing output table logic into clean DataFrames.
- **Native / Cascaded Python Dependencies**:
  - `numpy`, `scipy` (specifically `ndimage` and `stats`), `cv2` (OpenCV).

## 4. Hardcoded Assumptions
- **Node Naming Assumptions**: The logic heavily expects an exact segmented part named `aorta` (case-sensitive) to remain after the extraction.
- **Path Auto-Creation**: During the Batch Inference phase (`BatchInferenceButton`), it will create a folder appended with `_AortaQuanBatchResult` directly alongside your selected root data directory to store the outputs formatted as `.mrb`.
- **Sample Data Linkage**: It natively registers `DeepAorta1` and `DeepAorta2` URLs to the GitHub `SlicerTestingData` repo.
